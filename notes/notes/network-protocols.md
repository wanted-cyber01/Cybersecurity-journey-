 🌐 Network Protocols (Basics)

 📌 What is a Network Protocol?
A network protocol is a set of rules that allows devices to communicate over a network.



 🌍 Common Network Protocols

 🌐 HTTP / HTTPS
- HTTP → Used for websites (not secure)
- HTTPS → Secure version (encrypted)
- Ports:
  - 80 → HTTP
  - 443 → HTTPS



  📂 FTP (File Transfer Protocol)
- Used to transfer files between systems
- Port: 21



  🔋 SMTP (Simple Mail Transfer Protocol)
- Used to send emails



  📬 POP3 / IMAP
- Used to receive emails
- POP3 → downloads emails
- IMAP → syncs emails



  🌍 DNS (Domain Name System)
- Converts domain names to IP addresses
- Example:
  google.com → IP address



  ⚙️ DHCP (Dynamic Host Configuration Protocol)
- Automatically assigns IP addresses to devices



  🔐 SSH (Secure Shell)
- Secure remote login to a system
- Port: 22



  🔄 TCP/IP
- TCP → reliable communication
- IP → handles addressing



  ⚡ UDP (User Datagram Protocol)
- Faster but less reliable than TCP
- Used in streaming and gaming



  🔗 ARP (Address Resolution Protocol)
- Maps IP address to MAC address



 📡 ICMP (Internet Control Message Protocol)
- Used for network diagnostics
- Example: ping



  ⏰ NTP (Network Time Protocol)
- Synchronizes time across systems



  🖥️ Telnet
- Remote access (not secure)



  📊 SNMP (Simple Network Management Protocol)
- Used to monitor network devices



  🌐 Routing Protocols (RIP & OSPF)
- Help routers find best path for data



  🧠 What I Learned
- Protocols define how communication happens on networks
- Different protocols serve different purposes
- Ports are linked to specific protocols
- Secure protocols (HTTPS, SSH) are important in cybersecurity
